Fix #1: Keyboard Shortcut Change FIXED

Previous Issue: Ctrl+S conflicted with browser's save page function
New Behavior: Changed to Ctrl+/ (Ctrl + Forward Slash)

Updated Keyboard Shortcuts

  • Ctrl+/ - Toggle saved searches dropdown
  • Ctrl+Shift+S - Save current search with name
  • Alt+S - Quick save current search
Test Steps:
  1. Start the application
  2. Click in the search bar or anywhere on the page
  3. Press Ctrl+/ (Windows/Linux) or Cmd+/ (Mac)
  4. Verify: Saved searches dropdown opens without browser save dialog
  5. Press Ctrl+/ again
  6. Verify: Dropdown closes
  7. Test Ctrl+S - should trigger browser save (no longer our shortcut)
Note: The shortcut works from anywhere in the app, including when the search bar has focus.

Fix #2: Deletion Error Resolution FIXED

Previous Issue: Using browser's confirm() dialog caused "Oops! Something went wrong" error
New Behavior: Custom confirmation dialogs for all destructive actions

Test Deleting a Single Search

Test Steps:
  1. Open saved searches with Ctrl+/
  2. Hover over any saved search item
  3. Click the delete (trash) icon
  4. Verify: Custom confirmation dialog appears (not browser confirm)
  5. Click "Cancel" - verify search is not deleted
  6. Click delete icon again
  7. Click "Delete" in the confirmation dialog
  8. Verify: Search is deleted and toast notification appears
  9. Verify: No "Oops!" error message appears

Test Clear All Searches

Test Steps:
  1. Open saved searches dropdown
  2. Click "Clear all searches" link at the bottom
  3. Verify: Custom confirmation dialog appears
  4. Verify: Dialog shows count of searches to be deleted
  5. Test both Cancel and Delete All buttons
  6. Verify: No error messages appear

Test Save Search Dialog

Test Steps:
  1. Enter some search terms
  2. Press Ctrl+Shift+S or click Save Current
  3. Verify: Custom save dialog appears (not browser prompt)
  4. Enter a name and press Enter or click Save
  5. Verify: Search is saved with toast notification
  6. Test Escape key to cancel
